# ⚔️ Java Swing + MySQL 放置型 RPG

> 我的第三個完整 Java 專題，也是目前規模最大的一個專案。

本專案以 **Java Swing** 開發桌面遊戲介面，搭配 **MySQL** 作為資料庫，並採用 **MVC 架構、DAO Pattern、Service Layer** 等設計方式完成。

與前兩個專題相比，這次不只是完成遊戲功能，更著重於程式架構、物件導向設計以及資料庫分層管理，希望讓專案更容易維護與擴充。

---

# 📖 專案介紹

本專案是一款放置型 RPG。

玩家可以建立角色、登入遊戲、選擇英雄，並透過自動戰鬥獲得經驗值與裝備，逐步培養自己的角色。

整個專案皆使用 Java 開發，並將畫面、商業邏輯及資料庫操作分離，使程式碼更具可讀性與維護性。

# 🏗 專案架構

```text
使用者操作
      │
      ▼
Java Swing（UI）
      │
      ▼
Controller
      │
      ▼
Service
      │
      ▼
DAO
      │
      ▼
MySQL
```

本專案將各層職責分開：

* UI 負責畫面與操作
* Service 負責遊戲邏輯
* DAO 負責資料庫存取
* Entity / DTO / VO 負責資料傳遞

這樣的設計讓程式更容易維護，也方便未來增加新功能。

# 🌱 開發心得

這個專案是我學習 Java 的第三份完整作品。

這個專案並不是我一個人全部構想出來的，其中有使用到Google Antigravity的幫助，遊戲設計、遊戲功能是以我的想法去做，而專案結構則是我和 AI 一同討論的結果。<br>

我經常思考一個方法或一段邏輯應該放在哪一層，才能符合各層的職責。對於某幾個MySQL資料表的欄位設計，也是經過討論後而得出的結果，當我遇到架構上的疑問時，我會與 AI 討論不同的設計方式，再理解各種方案的優缺點以及業界常見的做法，最後再決定採用哪一種。<br>

像是關於戰鬥系統的設計，由於戰鬥進行並不是單純的把資料寫入讀出修改，而是高頻率的運算，如果每一輪的戰鬥資訊都進行資料庫存取，不但效率較差，也會增加系統負擔。經過討論最後選擇先在 Service 層完成整個戰鬥流程，待戰鬥結束後再一次更新需要保存的資料。這些都是從我自己的專案中一步一腳印的寶貴經驗，而這份專案也會是我成為一個合格工程師重要的養分。<br>

其實我認為這份專案還有許多可以改進的地方，不過這是我第一次認真的思考一份專案中的軟體架構、物件導向以及分層設計的過程，所以我希望保留它最原本的樣子，讓未來的我看到這份專案時，能夠回想起自己從零開始，一步一步朝著成為一位合格軟體工程師努力的過程。
